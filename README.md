# MemoryLoader

`MemoryLoader` - это консольная утилита для эмуляции процесса, который активно занимает оперативную память.

Основной сценарий использования: проверить, как ваше основное приложение ведет себя, если в системе появляется другой процесс, начинающий забирать ОЗУ.

## Как работает программа

`MemoryLoader` выделяет память блоками через unmanaged allocation (`Marshal.AllocHGlobal`) и записывает данные в выделенные страницы памяти, чтобы операционная система действительно зафиксировала этот объем за процессом.

Поддерживаются два режима:

- `hold` - программа заполняет память до указанного объема и удерживает ее, пока вы не остановите процесс или не закончится таймер.
- `swing` - программа заполняет память до указанного объема, держит ее некоторое время, затем освобождает и повторяет цикл.

По умолчанию программа работает бесконечно, пока вы не остановите ее вручную. Если задать таймер, она завершится сама.

## Параметры запуска

Общий формат:

```powershell
MemoryLoader.exe [процент] [-mb MB] [-mode hold|swing] [-time сек] [-chunk MB] [-hold сек] [-release сек]
```

Или через `dotnet run`:

```powershell
dotnet run -- [процент] [-mb MB] [-mode hold|swing] [-time сек] [-chunk MB] [-hold сек] [-release сек]
```

### Поддерживаемые параметры

- `[процент]`
  Необязательный параметр.
  Задает, сколько процентов от всей физической RAM должен занять `MemoryLoader`.
  Диапазон: от `1` до `99`.
  Если указан параметр `-mb`, то используется `-mb`, а не процент.

- `-mb <число>`
  Задает целевой объем памяти в мегабайтах, который должен занять `MemoryLoader`.
  Пример: `-mb 2048`

- `-mode hold`
  Заполнить память и не освобождать ее.

- `-mode swing`
  Заполнять и освобождать память циклически.

- `-time <секунды>`
  Сколько секунд программа должна работать.
  Если не задан, программа работает до ручной остановки.

- `-chunk <MB>`
  Размер одного блока выделения памяти в мегабайтах.
  По умолчанию: `50`

- `-hold <секунды>`
  Только для режима `swing`.
  Сколько секунд держать память после достижения целевого объема.
  По умолчанию: `5`

- `-release <секунды>`
  Только для режима `swing`.
  Сколько секунд ждать после освобождения памяти перед следующим циклом.
  По умолчанию: `5`

- `help`, `-h`, `--help`, `/?`
  Показать справку.

## Примеры запуска

Занять 40% RAM и не освобождать:

```powershell
MemoryLoader.exe 40 -mode hold
```

Занять 2048 MB и держать 60 секунд:

```powershell
MemoryLoader.exe -mb 2048 -mode hold -time 60
```

Занимать и освобождать 1024 MB циклически:

```powershell
MemoryLoader.exe -mb 1024 -mode swing -hold 10 -release 3
```

Показать справку:

```powershell
MemoryLoader.exe help
```

## Сборка для разработки

Если на машине установлен .NET SDK, можно собрать проект командой:

```powershell
dotnet build
```

## Публикация готового EXE без установленного .NET

Для конечного запуска без установленного `.NET` используется `publish`, а не `build`.

Проект настроен на single-file self-contained publish. Это значит:

- на выходе получается один `exe`
- на целевой машине не нужен установленный `.NET`

### Сборка 64-битной версии

```powershell
dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o Redist\64
```

После этого готовый файл будет лежать здесь:

```text
Redist\64\MemoryLoader.exe
```

### Сборка 32-битной версии

```powershell
dotnet publish -c Release -r win-x86 --self-contained true -p:PublishSingleFile=true -o Redist\32
```

После этого готовый файл будет лежать здесь:

```text
Redist\32\MemoryLoader.exe
```

### Автоматическая публикация сразу в две папки

В проекте есть скрипт:

```powershell
.\publish.ps1
```

```powershell
.\powershell -ExecutionPolicy Bypass -File .\publish.ps1
```

Который собирает две версии автоматически:

- `Redist\64`
- `Redist\32`
